INTRODUCTION ~
  solidity-metrics.nvim is a thin Neovim wrapper around the
  `solidity-code-metrics` npm package. The in-editor view renders a native
  Neovim report with inline charts and graphs via image.nvim and Graphviz.

REQUIREMENTS ~
  - Neovim 0.9+
  - Node.js
  - image.nvim configured and working
  - Graphviz `dot`
  - Node.js
  - `solidity-code-metrics` available either as a project-local
    `node_modules/.bin/solidity-code-metrics`, in PATH, or via a custom
    command configured via |solidity-metrics-config|.

COMMANDS ~
  :SolidityMetricsFile
      Run metrics for the current buffer.

  :SolidityMetricsWorkspace [path]
      Run metrics for the detected workspace root or a supplied directory.

  :SolidityMetricsScope [file]
      Run metrics using a scope file. Defaults to `scope.txt` in the root.

  :SolidityMetricsHtml [output]
      Export the last request as HTML. Defaults to `solidity-metrics.html`
      in the request root.

NOTES ~
  Native charts and graphs are rendered inline inside Neovim via image.nvim.
  Visual report mode requires Graphviz `dot` and image.nvim.
  Use :SolidityMetricsHtml for the full exported HTML report.

CONFIGURATION ~
                                                        *solidity-metrics-config*
>
  require('solidity_metrics').setup({
    cmd = nil,
    prefer_local = true,
    local_bin = 'solidity-code-metrics',
    exclude = {
      '**/node_modules/**',
      '**/mock*/**',
      '**/test*/**',
      '**/migrations/**',
      '**/Migrations.sol',
    },
    output = {
      kind = 'split',
      split = 'botright new',
    },
  })
<

HEALTH ~
